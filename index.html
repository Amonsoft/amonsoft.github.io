We are primarily developing software for the mobile Ecosystem. our major focus is on the roll out strategy, which is through Mobile Smartphones. So we have set out to streamline the distribution of smartphones in a more convenient and comfortable way for our clients suing long term contracts.

Tools Used at Amonsoft

At Amonsoft we power our technology and products using some of the high class product on market.
We as a mobile development company, try as much as possible to utilize differnt tools to achieve efficency , Productivity, fluidity and comfort when working to ensure of products work on as many platforms as possible and reach as many people as possible.
We provide this comprehensive list of how we operate and the differnt tools we use to showcase what a small company can use and utilize to create high class top notch products.
  • Servers
  • Text-Editors
  • Web Browsers
  • Databses
  • Programming Languages
  • IDEs

Servers

We use many Servers at Amonsoft to run and test our products in House, and one of our major key factors when selecting a server ;
  • Type of project we are undertaking
  • Speed of the server
  • Ease in execution of the compiled code to the broswer
  • Disk space it occupies on the computers
  • Programming language we are using for the project

    • *These are not in order of priority

  • Bracket(IDE)inbuilt WebServer

    • This is used when runnimg Web based projects without any server side code yet, as when we are in design phase. This helps us to execute to the browsers faster and make changes in real time without fireing up the browser. The only problem is that , it only launches in Google Chrome, but hope for more support or customisation from the team at bracket, Adobe Systems

  • Xampp Server

    • We use Xampp to run full web applications , Exploiting its power to combine the UI pages , Scripts and databases. When a project is predominatly using Php scripts, its easy to setup since php compilers are inbuilt in xampp. We also use the PhpMyadmin database to interact with databases , using the CRUD methods , making the application come alive.

  • Node js Server

    • We started using Node ealier in 2015, when we wanted to test building of a product with a light weight server, and one whose language was so similar to the javascript we used in websites. Node helped us achieve that, wrinting servers in Javasrcipt and manipulating them . Node also gives us the opportunity to monitor our server behaviour, how its responding to requests, how many request are being recieved.
